# 评估评分表

> 项目最终里程碑评估使用。
> 日常工单自审使用工单内的三维度清单（完整性/正确性/一致性）。

---

## 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 架构合规 | 15% | 实现遵循 ARCHITECTURE.md 中的核心原则和规则 |
| 代码质量 | 15% | 命名规范、结构清晰、无重复代码 |
| 测试覆盖 | 12% | 关键路径有测试，边界条件有覆盖 |
| 文档完整 | 10% | TBD 已全部回填，文档与代码一致 |
| 错误处理 | 12% | 异常路径有处理，日志有记录 |
| 安全性 | 10% | 无硬编码密钥，输入有校验 |
| 性能 | 8% | 无明显性能瓶颈 |
| 可维护性 | 10% | 模块边界清晰，依赖方向正确 |
| 交付完整 | 8% | 所有工单已完成，功能清单全部通过 |

---

## 工单级检查项

### 按工单类型分组

#### 基础设施类工单

- [ ] 目录结构符合约定
- [ ] 依赖配置正确
- [ ] 构建/启动命令可用

#### 数据层工单

- [ ] 数据模型定义完整
- [ ] 迁移脚本可执行
- [ ] 索引和约束正确

#### 中间件工单

- [ ] 异常路径有测试
- [ ] 配置可从环境变量读取
- [ ] 日志级别可配置

#### 业务域工单

- [ ] 行为匹配规格描述
- [ ] 场景覆盖完整
- [ ] 接口签名与规格一致

#### 集成类工单

- [ ] 端到端流程可走通
- [ ] 配置项全部可配置
- [ ] 部署脚本可用

---

## 评分规则

| 分数 | 含义 |
|------|------|
| 9-10 | 优秀：无缺陷，可作为范例 |
| 7-8 | 良好：有少量改进空间 |
| 5-6 | 及格：有明显缺陷但可用 |
| 3-4 | 不及格：有严重缺陷 |
| 1-2 | 严重不足：需要重写 |
