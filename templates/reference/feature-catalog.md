## 常见架构原则参考

以下原则在多数项目中普遍适用，可在建议时提供给用户：

1. **分层依赖方向**：api → domain → services → infra，禁止反向
2. **外部服务抽象**：外部服务（LLM / 数据库 / 缓存 / 对象存储）通过 Provider 接口访问，不直接调用 SDK
3. **配置外部化**：所有配置从环境变量读取，禁止硬编码
4. **错误边界**：每层只处理自己能处理的异常，未知异常向上抛出
5. **接口契约**：对外暴露的 API 有明确的请求/响应类型定义
